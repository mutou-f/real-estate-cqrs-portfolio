# Interview Notes: Auth

## 一言説明

`JWT を使った認証と、ロールベースの認可を入れています`

## このポートフォリオでの制御

- `APPLICANT`: 自分の申込を作成・提出できる
- `REVIEWER`: 申込を承認・却下できる
- `ADMIN`: 全体管理ができる

## 面談での回答例

`認証は JWT ベースで、ログイン後は access token で API を叩く構成にしています。認可は role を見て制御していて、申込者は作成と提出、審査担当は承認と却下ができるように分けています`

## 話す時の注意

- `認証` と `認可` を混ぜない
- 認証は `本人確認`
- 認可は `何ができるかの制御`
