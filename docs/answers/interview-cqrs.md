# Interview Notes: CQRS

## 一言説明

`CQRS は、更新処理と参照処理で責務を分ける考え方です`

## このポートフォリオでの使い方

- Command
  - 申込作成
  - 申込提出
  - 承認
  - 却下
- Query
  - 申込一覧
  - 申込詳細
  - 履歴表示

## 面談での回答例

`このポートフォリオでは、申込の作成や承認のような更新系は command として分け、一覧や詳細表示のような参照系は query として分けました。更新時はイベント保存と read model 更新までを行い、画面表示は query 側から read model を返す形にしています`

## 話す時の注意

- `常に CQRS が必要` とは言わない
- `更新と参照で要件が違う時に有効` と言う
