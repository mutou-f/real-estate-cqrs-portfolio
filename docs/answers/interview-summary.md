# Interview Summary

## 30秒版

`不動産申込・審査システムを題材に、JWT 認証、CQRS、Event Sourcing を組み合わせたポートフォリオを作っています。更新系は command、参照系は query で分け、状態変更は event store に保存し、画面表示は read model を使う構成にしています。認証と権限制御も入れているので、申込者と審査担当で扱える操作が変わるようにしています`

## 60秒版

`面談で話せるレベルまで理解を深めるために、不動産申込・審査システムのポートフォリオを作っています。JWT 認証でログインし、申込者・審査担当・管理者のロールで操作を分けています。CQRS の考え方としては、申込作成や承認のような更新系を command として分け、一覧や詳細表示は query 側で read model を返す形にしています。さらに Event Sourcing の理解のために、作成・提出・承認・却下といった状態変更を event store に保存して、履歴と監査性も確認できる構成にしています。実務で大規模運用したというより、採用しどころと複雑性を自分で理解するためのポートフォリオです`
