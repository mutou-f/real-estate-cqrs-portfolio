# Interview Summary

## 30秒版

`不動産申込・審査システムで、JWT 認証、CQRS、Event Sourcing を組み合わせた構成です。更新系は command、参照系は query で分け、状態変更は event store に保存し、画面表示は read model を使う構成にしています。認証と権限制御も入れているので、申込者と審査担当で扱える操作が変わるようにしています`

## 60秒版

`不動産申込・審査システムで、JWT 認証によりログインし、申込者・審査担当・管理者のロールで操作を分けています。CQRS の考え方として、申込作成や承認のような更新系を command として分け、一覧や詳細表示は query 側で read model を返す形にしています。さらに、作成・提出・承認・却下といった状態変更を event store に保存して、履歴と監査性も確認できる構成にしています`
